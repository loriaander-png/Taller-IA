¿Qué es un commit y por qué es importante?

Breve explicación y ejemplo de uso en Git

Un commit es una captura (snapshot) del estado del proyecto guardada en el historial del sistema de control de versiones (por ejemplo, Git), acompañada de un mensaje que describe qué se cambió y por qué.

Por qué es importante:

  • Mantiene un historial de cambios consultable y reversible.
  • Permite saber quién hizo cada cambio y cuándo (trazabilidad).
  • Facilita colaboración y revisiones de código (pull requests / code review).
  • Permite deshacer errores volviendo a commits anteriores.
  • Mejora la organización si los commits son atómicos y con mensajes claros.

Ejemplo básico de uso en Git:

git add .
git commit -m "Arregla validación de formulario: maneja campos vacíos"

Consejo breve: hacer commits pequeños y con mensajes descriptivos para facilitar lectura y revertir cambios si es necesario.
